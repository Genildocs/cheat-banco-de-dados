SQL vs NoSQL

Não é bicho de sete cabeças!

Entenda como os dois sistemas funcionam de forma simples e direta. Firebase, MongoDB, PostgreSQL, Supabase são apenas ferramentas que usam esses sistemas.

🧠 Como cada sistema pensa

🗃️

SQL

Pensa em: Tabelas com colunas e linhas, como uma planilha do Excel. Cada linha é um registro, cada coluna é um campo. Tudo organizadinho e estruturado.

Ferramentas que usam SQL:

PostgreSQL
MySQL
SQLite
Supabase
🍃

NoSQL

Pensa em: Documentos flexíveis, como arquivos JSON. Cada documento pode ter campos diferentes. É como ter uma pasta com vários tipos de arquivo.

Ferramentas que usam NoSQL:

MongoDB
Firebase
CouchDB
DynamoDB

🔧 Operações básicas (é mais simples do que parece!)

🔍 Buscar dados

SELECT * FROM usuarios 
WHERE idade > 18;

"Me dê todos os usuários que têm mais de 18 anos"

🔍 Buscar dados

db.usuarios.find({ 
  idade: { $gt: 18 } 
});

"Me dê todos os usuários que têm mais de 18 anos"

➕ Adicionar dados

INSERT INTO usuarios (nome, idade) 
VALUES ('João', 25);

"Adicione um novo usuário chamado João com 25 anos"

➕ Adicionar dados

db.usuarios.insertOne({ 
  nome: 'João', 
  idade: 25 
});

"Adicione um novo usuário chamado João com 25 anos"

✏️ Atualizar dados

UPDATE usuarios 
SET idade = 26 
WHERE nome = 'João';

"Mude a idade do João para 26 anos"

✏️ Atualizar dados

db.usuarios.updateOne(
  { nome: 'João' }, 
  { $set: { idade: 26 } }
);

"Mude a idade do João para 26 anos"

🗑️ Deletar dados

DELETE FROM usuarios 
WHERE idade < 18;

"Remova todos os usuários menores de 18 anos"

🗑️ Deletar dados

db.usuarios.deleteMany({ 
  idade: { $lt: 18 } 
});

"Remova todos os usuários menores de 18 anos"

💡 A verdade sobre bancos de dados

SQL não é bicho de sete cabeças. É só uma forma organizada de guardar dados, como uma planilha.

NoSQL não é "mais fácil". É só uma forma diferente de guardar dados, como documentos.

Ambos são ferramentas. A escolha depende do que você está construindo, não do que está na moda.

Comece com o que faz sentido para seu projeto. Você sempre pode migrar depois.
